Your task is to create simple, easily readable and understandable list of features offered by Klabis for business stakeholders. 
Primary use for this file is allowing business stakeholders do fast review of application functions

File will be created in docs/KlabisRequirements.html

1. find all specifications from openspec files
2. create list of current application requirements and functions. For every requirement, create short summary from scenarios defined for that requirement in specification file. 
   - drop any technical details, information should be strictly business oriented
3. check if target file already exists, if yes, delete it. 
4. create new file with the requirements overview 
   - it should be single HTML file with appropriate styling and nicely organized list of requirements with short description
   - file must be written in Czech language
