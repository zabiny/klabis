Scale-In Animation

Scale-in animation (used for modals)

@keyframes fadeInScale {
  from {
    opacity: 0;
    transform: scale(0.95);
  }
  to {
    opacity: 1;
    transform: scale(1);
  }
}
Animation Duration: 300ms cubic-bezier(0.4, 0, 0.2, 1)

Interaction Guidelines

  • Keep animations under 300ms for immediate feedback
  • Use consistent easing across all animations
  • Apply transitions to state changes (hover, focus, active)
  • Don't animate multiple elements simultaneously
  • Avoid animations on every interaction - be selective
  • Test animations for performance on lower-end devices

Best Practices for New Components

‚úÖ DO

  • Use semantic color names (primary, surface-base, border)
  • Maintain spacing using the 4px scale (2, 4, 8, 12, 16, 24, 32, 48px)
  • Apply transitions to all state changes
  • Test keyboard navigation for all interactive elements
  • Use Tailwind utility classes, avoid custom CSS
  • Include focus rings (ring-accent) on all interactive elements
  • Follow existing component patterns
  • Test in both dark mode and light mode

‚ùå DON'T

  • Use hardcoded hex values (#f44336, etc.)
  • Mix Tailwind utilities with inline styles
  • Create custom animations without adding to theme
  • Remove or hide focus states
  • Use color alone to convey meaning
  • Animate too many things at once
  • Break the spacing scale with arbitrary values
  • Use generic font stack (always use Sohne/Geist/Fira Code)
