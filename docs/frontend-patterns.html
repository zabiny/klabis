Frontend UI Patterns

Katalog opakujících se vzorů pro UI komponenty nad filtry a seznamy. Vzory jsou usazené v produkčním kódu (events, members); tento dokument je fixuje a slouží jako reference pro další filtr-bary. Udržuje se krátký a živý — přidávejte další vzory, jakmile se opakuje nějaký nový.

Filter bar layout

Filter bar je vodorovný kontejner nad tabulkou. Všechny ovládací prvky (fulltext input, pill-groupy, binární toggly) jsou zarovnané do jedné řady s flex-wrapem pro úzké viewport. 

<div className="flex flex-wrap items-center gap-3 p-3 bg-surface-raised rounded-md border border-border">
    {/* fulltext + pill group + toggle */}
</div>

Kde najdeš příklady: EventsFilterBar.tsx, MembersFilterBar.tsx.

Fulltext search input (shared)

src/components/UI/FulltextSearchInput.tsx — jediná sdílená komponenta pro fulltextové hledání napříč listy. Pravidla:

  • Lokální state pro input (okamžitá vizuální odezva).
  • Debounce 250 ms před zápisem do URL.
  • Minimum 2 znaky po trimu; kratší dotaz odstraní query param z URL.
  • Jediný zdroj pravdy je URL query param (default q; název lze přepsat přes paramName).
  • Synchronizace při externí změně URL (back/forward, odkaz) — hodnota inputu se aktualizuje.
  • Placeholder i aria-label přichází jako props; žádné hardcoded stringy.
<FulltextSearchInput
    paramName="q"
    placeholder={labels.membersFilter.searchPlaceholder}
    ariaLabel={labels.membersFilter.search}
/>

Pill group (3-valued exclusive selector)

Tři vzájemně výlučné hodnoty (např. Aktivní / Neaktivní / Vše; Budoucí / Proběhlé / Vše). Preferovaný ovládací prvek namísto dropdownu nebo zaškrtávátka — je diskrétní, symetrický a přímo ukazuje všechny dostupné stavy.

  • ARIA: kontejner má role="group" + aria-label; aktivní pill nese aria-pressed="true".
  • Vizuál: aktivní pill má bg-primary text-white, ostatní bg-surface text-text-primary.
  • URL binding: klik píše hodnotu enumu (ACTIVE / INACTIVE / ALL) do příslušného query paramu.
  • Výchozí hodnota: injectuje se na prvním renderu přes setSearchParams(..., {replace: true}), ne jako defaultní prop.

Binary toggle (authority-gated)

Binární přepínač viditelný jen pro uživatele s určitou autoritou (např. checkbox "Moje přihlášky" jen pro uživatele s memberId; status pill group v members jen pro MEMBERS:MANAGE).

  • Gate se odvozuje z HAL signálu (přítomnost template/linku), ne z parseování JWT. Např. resourceData?._templates?.registerMember implikuje MEMBERS:MANAGE.
  • Když je skryto, backend API musí zapsaný parametr silent-ignorovat (ne 400/403), aby URL tampering nepřepadl na error stránku.

Default filter injection

"Užitečný default" — při prvním renderu stránky bez query paramů se výchozí hodnota zapíše do URL přes setSearchParams(prev => ..., { replace: true }). Použij useRef flag defaultAppliedRef, aby se efekt spustil jen jednou. 

const defaultAppliedRef = useRef(false);
useEffect(() => {
    if (defaultAppliedRef.current) return;
    defaultAppliedRef.current = true;

    if (!searchParams.get('status')) {
        setSearchParams((prev) => {
            const next = new URLSearchParams(prev);
            next.set('status', 'ACTIVE');
            return next;
        }, { replace: true });
    }
}, []); // eslint-disable-line react-hooks/exhaustive-deps

Empty-state copy

Pokud filtr/kombinace filtrů nevrátí žádné řádky, HalEmbeddedTable umí zobrazit emptyMessage. Používej generický text:

Žádní členové neodpovídají zadaným filtrům.
Žádné akce neodpovídají zadaným filtrům.

Kontextuální varianty podle typu filtru nejsou (zatím) potřeba — generický text stačí.

URL query params = single source of truth

Všechny filtry žijí v URL. Reload, back/forward a sdílení odkazu fungují out-of-the-box bez session storage ani server-side per-user preferencí. Komponenta filtr-baru čte stav přes useSearchParams a zapisuje přes setSearchParams. API volání bere stejné hodnoty z URL.
