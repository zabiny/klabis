Bezpečnost

Klabis je OAuth2 Authorization Server i Resource Server v jedné aplikaci. Frontend získává access token přes Authorization Code Grant + PKCE. Autorizace má tři vrstvy: method-level (@HasAuthority), endpoint-level (SecurityConfiguration) a field-level (@OwnerVisible + @OwnerId).

Přehled bezpečnostní vrstvy

TřídaÚčel
SecurityConfigurationHlavní filter chain — JWT validace, endpoint matchers, password encoder.
AuthorizationServerConfigurationSetup Spring Authorization Serveru (issuer, JWK, token endpointy).
AuthorizationServerCustomizer (interface)SPI bod pro přizpůsobení AS chainu — moduly mohou registrovat @Component.
KlabisJwtAuthenticationConverterZ JWT vyrobí KlabisJwtAuthenticationToken s typovaným UserId + memberIdUuid.
KlabisJwtAuthenticationTokenVlastní Authentication typ s getUserId(), getMemberIdUuid(), hasAuthority(Authority).
HasAuthorityMethodInterceptorAuthorizationAdvisor vyhodnocující @HasAuthority + @OwnerVisible.
FieldSecurityJacksonModuleJackson modul, který při serializaci response filtruje pole bez oprávnění.
SpaFallbackFilterServlet filtr (@Order(LOWEST_PRECEDENCE)) — pro browser navigaci na SPA cestu, která neodpovídá žádnému static resource ani backendovému handleru, forwarduje na /index.html. Patří k routingu, ne k autorizaci, ale tvoří poslední článek serving chainu pod security vrstvami.

HTTP request flow

Když browser žádá URL na backendu, request prochází několika vrstvami v tomto pořadí:

  1. Spring Security filter chains (řazené podle @Order) — autorizační server matcher (@Order(1)), /login form chain (@Order(2)), /docs/** session-auth chain (@Order(3)), /h2-console/** chain (@Order(2), dev only), SPA text/html chain (@Order(4)), API + actuator + Swagger chain (@Order(5)). První, jehož securityMatcher sedí, request zpracuje; ostatní jsou přeskočeny.
  2. DispatcherServlet mapuje request na @Controller / @RequestMapping handler. Pokud najde, zavolá ho.
  3. ResourceHttpRequestHandler (přes WelcomePageHandlerMapping) hledá soubor v classpath:/static/. /index.html, /assets/**, /docs/index.html, /swagger-ui/index.html, /silent-renew.html, ikony — všechno odsud.
  4. SpaFallbackFilter (OncePerRequestFilter, LOWEST_PRECEDENCE) — teprve když všechny předchozí kroky selhaly:
    • Pokud request je GET, klient akceptuje text/html a cesta není na vyloučeném prefixu (/api/, /swagger-ui/, /v3/api-docs, /docs/, /oauth2/, /.well-known/, /actuator/, /h2-console/, /error) → forward na /index.html (SPA shell).
    • Jinak request projde dál a klient dostane standardní 404 (JSON pro API klienty, HTML error page jinak).
    Filtr rozhoduje předem: kontroluje resourceLoader.getResource("classpath:/static" + path).exists() a pokud soubor existuje, jen propustí request dál — žádné wrappování response.
Rule of thumb: SPA shell (/index.html) se vrací pouze když žádný předchozí handler request neobsloužil. Cokoli s vyhrazeným handlerem (Swagger UI, OpenAPI, dev manuál, OAuth2 endpointy, silent-renew, statické assety, API) nikdy nedostane SPA shell.
PWA service worker: ve frontendu (vite.config.ts) je nakonfigurovaný Workbox service worker s NavigationRoute, který v běžícím prohlížeči zachytává navigační requesty na SPA scope a vrací cached /index.html. Jeho navigateFallbackDenylist musí pokrývat stejné prefixy jako SpaFallbackFilter.EXCLUDED_PREFIXES + EXCLUDED_EXACT, jinak SW přepíše backend i pro URL, které backend obsluhuje (Swagger UI, dev manuál apod.).

JWT a vlastní claims

Access token vydávaný Klabisem nese kromě standardních claims (sub, aud, iat, exp) i:

  • user_id — UUID interního Useru (nikoli registračního čísla).
  • memberIdUuid — UUID členského profilu (chybí u admin účtů bez Member profilu).
  • authorities — pole stringů ve formátu "MEMBERS:MANAGE".
  • registrationNumber — používá se pro audit (createdBy/lastModifiedBy).

Konstanty názvů jsou v KlabisOAuth2ClaimNames. Jejich přidání do tokenu zajišťuje KlabisAuthorizationServerCustomizer v modulu members (viz sekce "OAuth2 přizpůsobení" níže na této stránce).

KlabisJwtAuthenticationToken

public class KlabisJwtAuthenticationToken extends JwtAuthenticationToken {
    public UserId getUserId();
    public Optional<UUID> getMemberIdUuid();
    public boolean hasMemberProfile();
    public boolean hasAuthority(Authority authority);
    public String getUsername();
}
Pokud potřebujete v controlleru aktuálního uživatele, nepoužívejte přímo SecurityContextHolder. Použijte parametrové anotace @ActingUser / @ActingMember z modulu members (kap. 05) — ty automaticky resolvují z této tokenové třídy.

Authority enum

Type-safe seznam všech autorit. Eliminuje překlepy a umožňuje IDE refactoring. 

public enum Authority {
    CALENDAR_MANAGE("CALENDAR:MANAGE", Scope.CONTEXT_SPECIFIC),
    MEMBERS_MANAGE("MEMBERS:MANAGE", Scope.CONTEXT_SPECIFIC),
    MEMBERS_READ("MEMBERS:READ", Scope.CONTEXT_SPECIFIC),
    MEMBERS_PERMISSIONS("MEMBERS:PERMISSIONS", Scope.GLOBAL),
    EVENTS_READ("EVENTS:READ", Scope.GLOBAL),
    EVENTS_MANAGE("EVENTS:MANAGE", Scope.CONTEXT_SPECIFIC),
    GROUPS_TRAINING("GROUPS:TRAINING", Scope.GLOBAL);

    public String getValue();           // "MEMBERS:MANAGE"
    public Scope getScope();            // GLOBAL nebo CONTEXT_SPECIFIC
    public static Authority fromString(String value);
    public static Set<Authority> getStandardUserAuthorities();
    public static boolean isKnownAuthority(String value);
}
Scope.GLOBAL
Autorita platná pro celou aplikaci, nelze ji udělit přes group membership (admin-level).
Scope.CONTEXT_SPECIFIC
Lze (do budoucna) udělit přes group membership v daném kontextu.
isKnownAuthority(value)
Použije se k filtrování — Spring DaoAuthenticationProvider přidává FACTOR_PASSWORD autoritu, kterou nechceme vidět v JWT.

Method-level autorizace: @HasAuthority

@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
public @interface HasAuthority {
    Authority value();
}

Použití na metodě nebo třídě (metoda overriduje třídu):

@RestController
class MemberController {

    @HasAuthority(Authority.MEMBERS_READ)
    @GetMapping("/members/{id}")
    public ... getMember(@PathVariable MemberId id) { ... }

    @HasAuthority(Authority.MEMBERS_MANAGE)
    @OwnerVisible
    @PatchMapping("/members/{id}")
    public ... updateMember(@PathVariable @OwnerId MemberId id, ...) { ... }
}
Při kombinaci s @OwnerVisible je vyhodnocení OR: přístup povolen, pokud má uživatel autoritu nebo je vlastníkem (parameter s @OwnerId).

Field-level autorizace

Pole DTO se filtrují podle autorizace při serializaci. Anotace lze kombinovat (OR semantika). Field bez oprávnění se buď vynechá z odpovědi, nebo nahradí null/maskovanou hodnotou (záleží na konfiguraci handleru). 

public record MemberDetailsResponse(
        UUID id,
        String firstName,
        @HasAuthority(Authority.MEMBERS_MANAGE) @OwnerVisible
        String email,
        @HasAuthority(Authority.MEMBERS_MANAGE) @OwnerVisible
        BirthNumber birthNumber,
        @OwnerId UUID ownerId
) {}
@OwnerVisible
Pole je viditelné pro vlastníka. Vlastník je určen polem označeným @OwnerId v stejném recordu (nebo automaticky, pokud existuje právě jedno pole konvertibilní na UUID).
@OwnerId
Označuje, které pole/parametr nese identifikátor vlastníka. Hodnota se porovnává proti memberIdUuid z aktuálního JWT tokenu.
@HasAuthority + @OwnerVisible
OR semantika — přístup, pokud je splněna alespoň jedna podmínka.

OwnershipResolver

Strategy interface, který určí, zda je aktuální uživatel vlastníkem objektu. Výchozí implementace (DefaultOwnershipResolver) bere hodnotu z @OwnerId a porovnává s memberIdUuid z KlabisJwtAuthenticationToken. Konverze typu na UUID probíhá přes Spring ConversionService — proto např. modul members registruje MemberIdToUuidConverter.

Bean OwnershipResolver je lazy resolvován z ApplicationContext uvnitř HasAuthorityMethodInterceptor. Eager injection způsobí "No ServletContext set" při startu (security advisor se inicializuje dříve než web kontext).

OAuth2 přizpůsobení v modulu members

Modul members rozšiřuje sdílený OAuth2 stack o bridging specifický pro Klabis mezi User aggregate a Spring Security. Tyto třídy jsou framework-úrovňové, ne byznys logika.

KlabisAuthorizationServerCustomizer
Implementace AuthorizationServerCustomizer SPI z common.security. Zapojuje token claims specifické pro Klabis (registrationNumber, memberIdUuid, firstName, lastName, authorities) do AS chainu během vydávání tokenu. Auto-registruje se jako @Component.
KlabisUserDetailsService
Bridge: načte User + UserPermissions a vrátí Spring Security UserDetails. Klíčové pro DaoAuthenticationProvider při login flow. Poskytuje i metodu loadKlabisUserDetails(userName) pro přístup z aplikačního kódu nezávislý na Spring Security.
MemberIdToUuidConverter
Spring Converter<MemberId, UUID> registrovaný do ConversionService. Vyžadovaný pro field-level autorizaci — DefaultOwnershipResolver potřebuje generic konverzi @OwnerId hodnoty na UUID, aby ji mohl porovnat s memberIdUuid z JWT.
