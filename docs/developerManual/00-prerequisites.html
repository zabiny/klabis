Předpoklady

Tato stránka je rozcestník pro vývojáře, kteří se potřebují rychle zorientovat v technologiích používaných v Klabisu. Každá sekce dává pětivětný přehled a odkaz na oficiální dokumentaci. Pokud vám něco zní povědomě, klidně přeskočte.

Java 21 a Spring Boot 4

Backend běží na Javě 21+ a Spring Bootu 4.0.x. Spring Boot je opinionated kontejner pro Spring aplikace se startovacími závislostmi, auto-konfigurací, embedded serverem a "production-ready" nástroji (actuator, health checky). Konfigurace se píše do application.yml nebo přes proměnné prostředí.

Spring Boot Reference · Java 21 docs

Domain-Driven Design (DDD) — taktické patterny

Klabis používá taktické DDD building blocks: Aggregate Root (kořen konzistenčního celku), Entity (objekt s identitou), Value Object (immutable hodnota bez identity), Domain Event (fakt, který se v doméně stal) a Repository (kolekce-like přístup k agregátům). Cílem je modelovat byznys jazykem (ubiquitous language) a držet invariants uvnitř agregátu.

DDD Reference (Eric Evans, PDF) · DDD Aggregate (Fowler)

Hexagonální architektura (Ports & Adapters)

Doménová logika je v "hexagonu" obklopeném porty (interfaces) a adaptéry (implementace). Primární porty (in) jsou volány UI/REST, sekundární porty (out) jsou volány doménou (typicky persistence). Klabis to kombinuje s jMolecules anotacemi (@PrimaryPort, @SecondaryPort, @PrimaryAdapter, @SecondaryAdapter) a fyzicky to dělí do package domain / application / infrastructure.

Hexagonal Architecture (Cockburn) · jMolecules

Spring Modulith

Modulární monolit organizovaný do application modules (kořenové package pod com.klabis). Spring Modulith vynucuje pravidla závislostí mezi moduly, publikuje domain eventy přes transactional outbox (tabulka event_publication) a podporuje @ApplicationModuleTest pro izolované testování modulu. 

@ApplicationModuleListener
public void on(MemberCreatedEvent event) { … }

Spring Modulith Reference

Spring Data JDBC

Lehký ORM postavený na agregátu jako jednotce perzistence. Bez lazy loadingu, bez session, bez change-trackingu — při save() se ukládá celý agregát. Mapování přes konvence + JdbcCustomConversions pro vlastní typy. V Klabisu se ukládá přes Memento pattern: doménový agregát se konvertuje do persistence-only recordu (memento), repository ukládá memento.

Spring Data JDBC Reference

Spring Security a OAuth2

Aplikace funguje současně jako Authorization Server (vydává JWT tokeny) i jako Resource Server (validuje vlastní tokeny). Frontend je SPA klient, který získává access token přes Authorization Code Grant + PKCE. Method security se zapíná @EnableMethodSecurity, autorizace se vyhodnocuje přes AuthorizationManager a AuthorizationAdvisor.

Spring Security Reference · Spring Authorization Server · OAuth2 RFC 6749

Spring HATEOAS, HAL, HAL+FORMS

REST API vrací JSON podle HAL (links + embedded resources). Klabis používá rozšíření HAL+FORMS (media type application/prs.hal-forms+json), které navíc popisuje affordances — tj. jak může klient s daným resourcem manipulovat (POST/PUT/PATCH/DELETE) včetně metadat o polích formuláře. Spring HATEOAS poskytuje linkTo(), methodOn(), RepresentationModelAssembler a Jackson serialization. 

EntityModel.of(dto)
    .add(linkTo(methodOn(MemberController.class).get(id)).withSelfRel())
    .add(Affordances.of(linkTo(…)).afford(HttpMethod.PATCH).withInput(MemberPatchDto.class).toLink());

Spring HATEOAS Reference · HAL spec · HAL+FORMS draft

Jackson

JSON serializace/deserializace přes Jackson. Klabis používá vlastní moduly: PatchFieldModule (pro PATCH semantiku), HalFormsMultiPropertyModule (rozšířená metadata) a FieldSecurityJacksonModule (filtrování polí podle autorizace).

Jackson docs

MapStruct

Compile-time generátor mapperů mezi DTO a doménovými typy. Mapper je interface s anotacemi @Mapper, @Mapping; implementaci generuje annotation processor do build/generated/. Bezpečnější a rychlejší než reflexní mappery.

MapStruct docs

Resilience4j, Jasypt, Flyway

Resilience4j
Knihovna pro fault tolerance — Klabis ji používá pro per-key rate limiting (viz kap. 09). docs
Jasypt
Šifrování řetězců (PBE) — používá se pro GDPR-citlivá pole (viz EncryptedString v kap. 02). docs
Flyway
SQL migrace s verzí. Klabis má jen 3 baseline skripty (V001 doménové tabulky, V002 OAuth2, V003 Modulith). Změny schématu se zapisují přímo do existujících scriptů (DB se v devu resetuje). docs

Frontend stack

Frontend je SPA v React 19 + TypeScript 5.8 postavená přes Vite 7. Klabis-specific patterny pro práci s HAL+FORMS odpověďmi popisuje kap. 11.

React + TypeScript + Vite
Bundler s HMR, dev server na portu 3000. React · Vite
TanStack Query (React Query)
Server-state management — cache, deduplication, refetching. Konvence: refetchOnWindowFocus: false, retry: 1, query keys jako ['members'], ['members', id]. docs
React Router DOM
Client-side routing. URL je single source of truth pro aktuální resource (HalRouteContext kombinuje pathname s HAL fetch). docs
Tailwind CSS
Utility-first styling. Klabis má vlastní theme tokens v src/theme/. docs
Formik + Yup
Form state + validační schémata. HAL+FORMS komponenty Formik používají interně — vlastní Formik formuláře potřebujete jen mimo HAL flow. Formik · Yup
oidc-client-ts
OAuth2/OIDC klient — Authorization Code Grant + PKCE. Wrapper v src/api/klabisUserManager.ts. docs
openapi-fetch + openapi-react-query
Typesafe HTTP klient nad TanStack Query, generovaný z OpenAPI specifikace (npm run openapi generuje src/api/klabisApi.d.ts). openapi-fetch
Vitest + Testing Library
Frontend testovací runner (jsdom environment). Testy v src/**/*.{test,spec}.{ts,tsx}. Vitest · Testing Library

Testovací stack (backend)

JUnit 5, AssertJ, Mockito a Spring Boot test slicy (@WebMvcTest, @DataJdbcTest, @SpringBootTest). Pro modulové testy @ApplicationModuleTest ze Spring Modulith. Pro integrace s reálnou databází TestContainers (PostgreSQL). Klabis-specific anotace @WithKlabisMockUser a @E2ETest najdete v kap. 10.
