Developer Manual

Klabis — Developer Manual

Praktický průvodce po "framework" části backendu Klabisu — sdíleném jádru, které zajišťuje autorizaci, perzistenci, HAL+FORMS API, eventy a další společné věci pro všechny moduly aplikace. Cílem manuálu je popsat, co je v projektu vlastní (specifické pro Klabis) a jak to používat při psaní nové funkcionality.

Pro koho je manuál určen

Manuál předpokládá, že čtenář umí Spring Boot, Spring Security, Spring Data, DDD a hexagonální architekturu. Pokud některou z těchto oblastí neznáte, začněte stránkou Předpoklady — obsahuje rychlý přehled s odkazy na oficiální zdroje.

Obsah

  1. 00 Předpoklady Rychlý přehled Spring Boot, Spring Modulith, Spring HATEOAS, Spring Data JDBC, Spring Security/OAuth2, DDD, hexagonální architektury.
  2. 01 Architektura projektu Modulární monolit, struktura packages (domain/application/infrastructure), pravidla závislostí, jMolecules.
  3. 02 Doménová vrstva KlabisAggregateRoot, AuditMetadata, type-safe IDs, value objects, domain events, EncryptedString.
  4. 03 Aplikační vrstva (services) Konvence pro @PrimaryPort services, sdílené UserService/PermissionService/PasswordSetupService, EncryptionService.
  5. 04 Perzistence Spring Data JDBC konfigurace, vlastní konvertory, Memento pattern, TranslatedPageable.
  6. 05 REST API a HAL+FORMS @HalForms, HalFormsSupport, link/affordance buildery, RootController, PatchField, validace, exception handling, @ActingUser/@ActingMember.
  7. 06 Bezpečnost OAuth2 AS+RS, JWT s vlastními claims, Authority enum, @HasAuthority, field-level (@OwnerVisible+@OwnerId), OAuth2 přizpůsobení v members.
  8. 07 Common — sdílené stavební bloky UserGroup + Invitation building block, EmailService, TemplateRenderer, hierarchie doménových výjimek.
  9. 08 Spring Modulith a moduly Domain eventy, @ApplicationModuleListener, transactional outbox, idempotence, observability metriky.
  10. 09 Cross-cutting concerns Bootstrap dat, MDC logging, PerKeyRateLimiter.
  11. 10 Testování Testovací nástroje specifické pro Klabis: @WithKlabisMockUser, @E2ETest, vlastní assertions, memento round-trip testy.
  12. 11 Frontend HAL+FORMS klient, useHalPageData, HalFormButton/HalFormsPageLayout, KlabisTable, přizpůsobení stránek, lokalizace.
  13. 12 Práce s Claude Code OpenSpec workflow, dostupné skills a agenti, jak přidat funkcionalitu, refaktoring, sandbox pravidla.
