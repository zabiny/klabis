Aplikační vrstva (services)

Application services jsou vstupní body do doménové logiky — orchestrují use cases, drží transakční hranici a koordinují více agregátů. V Klabisu je interface vždy @PrimaryPort v {module}/application/, implementace je @Service. Sdílené services pro práci s uživateli (UserService, PermissionService, PasswordSetupService) žijí v common.users. Tato kapitola také pokrývá šifrovací EncryptionService v common.encryption.

Konvence application services

Interface s @PrimaryPort
V {module}/application/, jméno odpovídá use casu (např. RegistrationPort, ManagementPort). Pro sdílené services v common stačí jméno {Subject}Service.
Inner record commands
Vstupní DTO patří jako vnořený record do interface (RegistrationPort.RegisterNewMember). Drží jen data z aplikační vrstvy — service sám doplní generované hodnoty (ID, registrační číslo).
@Transactional na metodě
Anotace patří na rozhraní (nebo metodu rozhraní), aby byla viditelná i pro AOP advisory bez proxování implementace.
Implementace bez prefixu/sufixu Impl
Pojmenujte podle role (RegistrationService), ne podle implementace. Mappery a privátní pomocníci patří do package infrastructure.
@PrimaryPort
public interface RegistrationPort {
    record RegisterNewMember(
            PersonalInformation personalInformation,
            Address address,
            EmailAddress email,
            UserId registeredBy
    ) {}

    @Transactional
    Member registerMember(RegisterNewMember command);
}

@Service
class RegistrationService implements RegistrationPort {
    private final UserService userService;
    private final MemberRepository memberRepository;
    // …
}
Application service nikdy nevrací přímo entity persistence vrstvy — vrací doménový agregát nebo explicitní DTO. Pro REST DTO konverzi slouží mapper v infrastructure/restapi/.

Sdílené user services (common.users)

Framework-úroveň services pro správu uživatelů. Exponovány jako @PrimaryPort v kořeni common.users, aby je mohly volat všechny moduly.

UserService

@PrimaryPort
public interface UserService {
    UserId createUser(String username, String email, Set<Authority> authorities);
    UserId createActiveUser(String username, String passwordHash, Set<Authority> authorities);
    Optional<User> findById(UserId id);
    Optional<User> findByUsername(String username);
    void suspendUser(UserId id);
    void resumeUser(UserId id);
}
createUser(...)
Vytvoří uživatele se statusem PENDING_ACTIVATION. Publikuje UserCreatedEvent, na který reaguje PasswordSetupService a pošle setup email.
createActiveUser(...)
Bootstrap varianta — uživatel rovnou ACTIVE s předpřipraveným hash hesla, bez emailu.

PermissionService

@PrimaryPort
public interface PermissionService {
    UserPermissions updateUserPermissions(UserId userId, Set<Authority> newAuthorities);
    UserPermissions getUserPermissions(UserId userId);
}

PasswordSetupService

@PrimaryPort
public interface PasswordSetupService {
    record SetupPasswordCommand(String token, String password) {}

    GeneratedTokenResult generateToken(User user);
    void sendPasswordSetupEmail(String firstName, String email, String plainToken);
    PasswordSetupToken validateToken(String plainToken);
    User completePasswordSetup(SetupPasswordCommand command, String ipAddress);
    void requestNewToken(String registrationNumber, String email);
}

AuthorizationQueryService

Read-only služba pro dotazy "má uživatel X autoritu Y v kontextu Z?". Neexponuje doménový stav, pouze rozhoduje. Vhodné pro adaptace v jiných modulech.

EncryptionService — šifrování mimo persistenci

Pokud potřebujete zašifrovat řetězec mimo JDBC vrstvu (např. před uložením do tokenu nebo do cache), je k dispozici EncryptionService v common.encryption. EncryptionConfiguration sestavuje Jasypt encryptor podle property klabis.encryption.password. 

String cipher = encryptionService.encrypt(plaintext);
String plain  = encryptionService.decrypt(cipher);
Pro běžné šifrování DB polí používejte hodnotový typ EncryptedString (kap. 02) — šifrování probíhá transparentně přes JDBC converters. EncryptionService je únikový poklop pro výjimečné scénáře.
