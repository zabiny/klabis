Perzistence

Klabis používá Spring Data JDBC s Memento patternem — doménový agregát se při ukládání převede do persistence-only recordu, který je čistě o mapování na DB. Domain typy tak nejsou znečištěné JDBC anotacemi a nemají závislost na Spring frameworku.

JdbcConfiguration

Centrální konfigurace JDBC vrstvy v common.jdbc. Aktivuje repository scanning, auditing a registruje custom converters. 

@Configuration
@EnableJdbcRepositories(basePackages = {"com.klabis"})
@EnableJdbcAuditing
@Import(EncryptionConfiguration.class)
public class JdbcConfiguration extends AbstractJdbcConfiguration {

    @Override
    public JdbcCustomConversions jdbcCustomConversions() { ... }

    @Bean
    public AuditorAware<String> auditorProvider();
}
AuditorAware<String>
Vrací přihlášeného uživatele (jeho registrationNumber ze SecurityContextu) pro audit pole. Bez autentikace vrací "system".
JdbcCustomConversions
Místo, kam se přidávají converter beany pro vlastní hodnotové typy (např. EncryptedString). Pro nový typ napište dva Convertery (sem ↔ String/UUID/…) a registrujte jako @Component; JdbcConfiguration je auto-discoverne (jsou injectované konstruktorem).

Memento pattern

Pro každý agregát existuje memento třída v {module}/infrastructure/jdbc/:

  • POJO s JDBC anotacemi (@Table, @Column, @Id, @Version).
  • Statická factory from(Aggregate) a metoda toAggregate().
  • Implementuje Persistable<ID> — agregát si sám určí, jestli je nový (isNew()).
  • Přeposílá domain eventy zpět na agregát přes @DomainEvents a @AfterDomainEventPublication.
@Table("members")
class MemberMemento implements Persistable<UUID> {

    @Id @Column("id") private UUID id;
    @Column("registration_number") private String registrationNumber;
    // … další pole …

    @Version private Long version;
    @CreatedDate private Instant createdAt;
    @CreatedBy   private String createdBy;
    @LastModifiedDate private Instant lastModifiedAt;
    @LastModifiedBy   private String lastModifiedBy;

    static MemberMemento from(Member member) { ... }
    Member toMember() { ... }

    @DomainEvents List<Object> events() { /* delegate to aggregate */ }
    @AfterDomainEventPublication void clear() { /* delegate to aggregate */ }
}

Repository pattern

Doménové repository interface je v {module}/domain/ (port). V infrastruktuře jsou dvě třídy:

{Aggregate}JdbcRepository (Spring Data interface)
Rozšiřuje CrudRepository<Memento, ID> nebo PagingAndSortingRepository. Sem se píší derived queries a @Query. Pracuje vždy s memento typem, ne s agregátem.
{Aggregate}RepositoryAdapter
Implementace doménového rozhraní (@SecondaryAdapter + @Repository). Konvertuje mezi mementem a agregátem, používá interní JdbcRepository a JdbcAggregateTemplate pro pokročilé dotazy.
@SecondaryAdapter
@Repository
class MemberRepositoryAdapter implements MemberRepository {

    private final MemberJdbcRepository jdbcRepository;
    private final JdbcAggregateTemplate jdbcAggregateTemplate;

    @Override
    public Member save(Member member) {
        MemberMemento savedMemento = jdbcRepository.save(MemberMemento.from(member));
        return savedMemento.toMember();
    }

    @Override
    public Optional<Member> findById(MemberId id) {
        return jdbcRepository.findById(id.uuid()).map(MemberMemento::toMember);
    }
}

TranslatedPageable — překlad sort properties

Doménové query API přijímá Pageable s doménovými property names (např. "firstName"). DB ale očekává sloupce (first_name). TranslatedPageable je dekorátor, který přeloží sort orders pomocí mapy. Neznámé properties prochází beze změny (fail-safe). 

private static final Map<String, String> DOMAIN_TO_DB_COLUMN = Map.of(
    "firstName", "first_name",
    "lastName", "last_name",
    "registrationNumber", "registration_number"
);

Pageable translated = TranslatedPageable.translate(originalPageable, DOMAIN_TO_DB_COLUMN);
return jdbcRepository.findAll(translated);

Custom queries s JdbcAggregateTemplate

Pro filtry s dynamickou kombinací podmínek se hodí JdbcAggregateTemplate + Query/Criteria z spring-data-relational: 

Criteria criteria = Criteria.empty();
if (filter.firstName() != null)
    criteria = criteria.and("first_name").like("%" + filter.firstName() + "%");

Query query = Query.query(criteria).with(translatedPageable);
List<Member> results = jdbcAggregateTemplate
        .findAll(query, MemberMemento.class).stream()
        .map(MemberMemento::toMember).toList();
