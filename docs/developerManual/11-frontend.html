Frontend

Klabis frontend je SPA postavená nad HAL+FORMS driveným klientem — backend posílá nejen data, ale i metadata o tom, jaké linky a formuláře jsou pro daný resource a uživatele dostupné. Frontend tato metadata interpretuje a generuje navigaci, akce a formuláře dynamicky. Tato kapitola popisuje Klabis-specific patterny: jak číst HAL response, jak postavit stránku, tabulku a formulář.

Generická znalost React, TypeScript, Vite, TanStack Query a oidc-client-ts je předpokládaná — viz kap. 00 Předpoklady. Tato kapitola pokrývá jen Klabis-specific kód.

Práce s HAL+FORMS response

Backend vrací HAL+FORMS strukturu: { ...data, _links, _embedded, _templates }. Pro typové utility z src/api/hateoas.ts: 

// Získat link podle rel name
const editLink: Link | null = getLink(resourceData, 'edit');
const href: string | null = getLinkHref(resourceData, 'edit');

// Načíst data z linku (následuje href)
const detail = await followLink<MemberDetail>(link);

// Získat embedded resources
const items = getEmbedded(resourceData, 'items');
Nikdy nehardcodujte URL. Všechna navigace probíhá přes _links a _templates z aktuálního resource. Pokud něco není v response, uživatel na to nemá oprávnění (backend filtruje field-level autorizací) — frontend se ani nemusí ptát.

useHalPageData — vstupní hook stránky

Většina stránek začíná zavoláním useHalPageData(). Hook automaticky načte HAL resource pro aktuální URL přes HalRouteContext a vrátí unified API: 

const useHalPageData<T extends HalResponse = HalResponse>(): UseHalPageDataReturn<T>;

interface UseHalPageDataReturn<T> {
    resourceData: T | null;
    isLoading: boolean;
    error: Error | null;
    isAdmin: boolean;

    route: {
        pathname: string;
        navigateToResource: (resource: HalResponse | Link) => void;
        refetch: () => Promise<void>;
        getResourceLink: (linkName?: string) => Link | null;
    };

    actions: { handleNavigateToItem: (href: string) => void; };

    // helpery
    getLinks(); getTemplates(); hasEmbedded(); getEmbeddedItems();
    isCollection(); hasLink(name); hasTemplate(name); hasForms();
}
export const MemberDetailPage = () => {
    const { resourceData, isLoading, error, isAdmin } = useHalPageData<MemberDetail>();

    if (isLoading) return <Spinner />;
    if (error)     return <ErrorDisplay error={error} />;
    if (!resourceData) return null;

    return <div>…</div>;
};
route.refetch() se automaticky volá v HalFormDisplay po úspěšném odeslání formuláře. Manuální volání potřebujete jen pro custom akce mimo HAL Forms.

HalSubresourceProvider — přepnutí kontextu na sub-resource

Stránky obvykle pracují s jedním HAL resource (ten, který odpovídá aktuální URL přes HalRouteProvider). Komponenty uvnitř ale nevědí, kde v HAL stromu jsou — sahají po datech přes useHalRoute() / useHalPageData(). Někdy ale potřebujete na části stránky dočasně přepnout kontext na jiný resource — typicky na sub-resource odkazovaný přes _links z aktuálního resource (majitel účtu, embedded kolekce transakcí, detail souvisejícího objektu apod.).

K tomu slouží HalSubresourceProvider (src/contexts/HalRouteContext.tsx). Přečte jmenovaný link z parent resource, udělá nad ním nový HalRouteProvider a vše uvnitř children pak pracuje s tímto sub-resource, jako by to byl hlavní route resource — bez změny URL v prohlížeči. 

interface HalSubresourceProviderProps {
    subresourceLinkName: string;   // jméno linku v parent resource._links
    children: React.ReactNode;
}

Předpokládejme, že stránka /members/123/finance načte HAL resource účtu v této podobě (zjednodušeno): 

{
    "balance": 1250,
    "_links": {
        "self":         { "href": "/api/members/123/finance" },
        "owner":        { "href": "/api/members/123" },
        "transactions": { "href": "/api/members/123/finance/transactions" }
    },
    "_templates": {
        "deposit": { "method": "POST", "target": "/api/members/123/finance/deposit", … }
    }
}

Linky owner a transactions ukazují na samostatné API endpointy — nejsou embedded v odpovědi. HalSubresourceProvider pro každý z nich udělá samostatný HTTP fetch a obsah zpřístupní uvnitř svého podstromu jako hlavní resource: 

export const MemberFinancePage = () => {
    const { resourceData, isLoading, route } = useHalPageData();
    if (isLoading) return <Skeleton />;

    return (
        <>
            <p>Zůstatek: {resourceData?.balance} Kč</p>

            {/* Fetch GET /api/members/123 — MemberName uvnitř vidí jako resourceData
                objekt majitele, ne účtu */}
            <HalSubresourceProvider subresourceLinkName="owner">
                <MemberName />
            </HalSubresourceProvider>

            {/* Fetch GET /api/members/123/finance/transactions — HalEmbeddedTable
                vidí kolekci transakcí jako svůj hlavní resource */}
            <HalSubresourceProvider subresourceLinkName="transactions">
                <HalEmbeddedTable collectionName="transactionItemResponseList" …>…</HalEmbeddedTable>
            </HalSubresourceProvider>
        </>
    );
};
Každý HalSubresourceProvider vyvolá samostatný GET na URL z odpovídajícího linku parent resource. Výsledek je cacheovaný React Query zvlášť podle cílové URL (stejná pravidla staleTime / gcTime jako u hlavního HalRouteProvider). Při třech sub-resource sekcích na stránce tedy vznikají tři nezávislé HTTP requesty — pokud by to byl výkonnostní problém, zvažte backend-side embedding místo linků.
Kdy použít
Když chcete uvnitř stránky použít existující HAL-aware komponentu (MemberName, HalEmbeddedTable, HalFormButton, …) proti jinému resource, než který visí na aktuální URL. Komponenta nemusí o přepnutí vědět — čte pořád přes useHalRoute().
Co se stane při chybějícím linku
Pokud parent resource nemá link daného jména (např. nedostatečná oprávnění, backend ho nevrátil), vykreslí se fallback text "Subresource … wasn't found". V praxi buď link podmiňte autorizací na backendu, nebo wrap obalte vlastní kontrolou přes getLink().
URL se nemění
Jde o čistě logické přepnutí kontextu v React stromu — window.location i route.pathname zůstávají na parent resource. Pro reálnou navigaci na sub-resource jako na samostatnou stránku použijte route.navigateToResource().
Vnořování
Poskytovatele lze vnořovat (sub-resource sub-resource), protože uvnitř se znovu použije HalRouteProvider. Data každého providera jsou cacheovaná React Query zvlášť podle cílové URL.
useHalRoute().getResourceLink() v main resource vrací jen linky z hlavního resource — linky ze sub-resource nevidí. Pokud potřebujete link ze sub-resource, čtěte ho uvnitř HalSubresourceProvider, kde je sub-resource vystavený jako hlavní.

Formuláře přes HAL+FORMS

Formuláře se generují z _templates v resource odpovědi. Klíčové komponenty v src/components/HalNavigator2/:

HalFormButton
Tlačítko pro otevření formuláře. Vrací null, pokud šablona neexistuje v _templates (= uživatel nemá oprávnění). Modal nebo inline režim.
HalFormsSection
Vyrendruje tlačítka pro všechny dostupné šablony. Ideální když chcete zobrazit "všechny dostupné akce" bez explicitního výčtu.
HalFormsPageLayout
Page wrapper, který orchestruje zobrazení formulářů — modal (z HalFormContext) i inline (z URL parametru ?form=…).
HalFormDisplay
Skutečný renderer formuláře. Obvykle se nepoužívá přímo — volají ho výše uvedené.
HalFormTemplateButton
Submit/cancel tlačítko uvnitř formuláře. Label se taháne přes getTemplateLabel.
MultiStepFormModal
Wrapper pro vícekrokové formuláře — viz MULTI_STEP_FORM.md.

Základní použití

<HalFormsPageLayout>
    <h1>Detail člena</h1>
    <HalFormButton name="updateMember" modal={true} />
    <HalFormButton name="suspendMember" modal={false} />
</HalFormsPageLayout>

Custom layout formuláře

Default layout vyrendruje pole pod sebou. Pro vlastní rozložení existují dva patterny — children s HalFormsFormField nebo callback s renderField(name): 

// Children pattern
<HalFormButton name="editMember" modal={true} customLayout={
    <div className="grid grid-cols-2 gap-4">
        <HalFormsFormField fieldName="firstName" />
        <HalFormsFormField fieldName="lastName" />
        <HalFormsFormField fieldName="submit" />
    </div>
} />

// Callback pattern
<HalFormButton name="editMember" modal={true} customLayout={(renderField) => (
    <>
        {renderField('firstName')}
        {renderField('lastName')}
        {renderField('submit')}
    </>
)} />
Pro inline formuláře nejde předat customLayout přes HalFormButton — místo toho použijte customLayouts prop na HalFormsPageLayout (mapa templateName → layout).

KlabisTable — tabulka pro HAL collection

Komponenta v src/components/KlabisTable/. Sloupce se deklarují přes <TableCell> children (extrakce ve renderu). Stav (paginace, sort) se kontroluje z parent komponenty. 

interface KlabisTableProps<T extends Record<string, unknown>> {
    data: T[];
    page?: TablePageData;
    error?: Error | null;
    onSortChange?: (column: string, direction: SortDirection) => void;
    onPageChange?: (page: number) => void;
    onRowsPerPageChange?: (size: number) => void;
    onRowClick?: (item: T) => void;
    currentPage?: number;
    currentSort?: SortState;
    children: ReactNode;            // <TableCell> deklarace
}
<KlabisTable
    data={items}
    page={pageInfo}
    currentPage={page}
    currentSort={sort}
    onPageChange={setPage}
    onSortChange={(col, dir) => setSort({ by: col, direction: dir })}
    onRowClick={(member) => navigate(toHref(getLink(member, 'self')!))}
>
    <TableCell column="registrationNumber" sortable>Reg. číslo</TableCell>
    <TableCell column="firstName" sortable>Jméno</TableCell>
    <TableCell column="email" hidden>Email</TableCell>
    <TableCell column="status" dataRender={({ value }) => (
        <Badge>{getEnumLabel('memberStatus', value as string)}</Badge>
    )} />
</KlabisTable>
column (povinné)
Klíč v datovém objektu. Použije se k zobrazení hodnoty a pro sort.
sortable
Zapíná řadicí šipky v hlavičce. Sort se předává nahoru přes onSortChange.
hidden
Sloupec není defaultně viditelný (uživatel ho může zapnout v column toggleru).
alwaysVisible
Sloupec nelze skrýt — užitečné pro klíčové identifikátory.
dataRender
Custom render buňky: ({ item, column, value }) => ReactNode. Bez ní se renderuje String(value).
Pro mobilní zařízení tabulka automaticky přepíná na CardView (každý řádek = card). Detekce je přes useMediaQuery.

KlabisTableWithQuery

Wrapper, který kombinuje KlabisTable s React Query / HAL fetchingem — vhodný pro stránky, kde nepotřebujete vlastní řízení dat. Podívejte se na implementaci, pokud chcete rovnou tabulku z HAL collection bez explicitního useQuery.

Customizace stránky

Klabis frontend podporuje dvě úrovně customizace:

Plně generované (GenericHalPage)
Stránka je vygenerována kompletně z HAL metadat — žádný custom kód, jen routing. Vhodné pro standardní CRUD obrazovky.
Customizovaná stránka
Vlastní layout, vlastní text, mix HAL komponent a custom kódu. Začíná useHalPageData(), končí v HalFormsPageLayout. Tomuto patternu se věnuje COMPONENTS.md v src/components/HalNavigator2/.

Šablona customizované stránky

export const EventDetailsPage = () => {
    const { resourceData, isLoading, error, route } = useHalPageData<EventDetail>();

    if (isLoading) return <Spinner />;
    if (error)     return <ErrorDisplay error={error} />;
    if (!resourceData) return null;

    return (
        <HalFormsPageLayout>
            <h1>{resourceData.name}</h1>
            <DetailRow label={labels.event.date} value={resourceData.date} />

            {/* Sekce s linky */}
            <HalLinksSection />

            {/* Akce — všechny dostupné šablony */}
            <HalFormsSection modal={true} />

            {/* Specifické tlačítko se zobrazí jen pokud má uživatel oprávnění */}
            <HalFormButton name="cancelEvent" modal={true} />
        </HalFormsPageLayout>
    );
};

Lokalizace

In-house systém v src/localization/labels.ts, žádný i18n framework (Klabis je jednojazyčný — čeština). Importujte z @/localization: 

import { labels, getFieldLabel, getTemplateLabel, getNavLabel, getEnumLabel } from '@/localization';

labels.member.firstName                                    // "Jméno"
getFieldLabel('eventDate')                                 // z labels.ts
getFieldLabel('eventDate', { eventDate: 'Custom popisek' }) // per-form override
getTemplateLabel('updateMember')                           // "Upravit"
getEnumLabel('memberStatus', 'ACTIVE')                     // "Aktivní"
getFieldLabel
Použije se v HalFormsInput pro popisky polí formuláře. Druhý argument umožňuje per-form override.
getTemplateLabel
Použije se v HalFormTemplateButton pro popisek submit tlačítka.
getNavLabel
Pro položky v hlavní navigaci (useRootNavigation).

Autentizace a fetch

klabisUserManager (src/api/klabisUserManager.ts)
oidc-client-ts wrapper. Authorization Code Grant + PKCE proti Klabis backendu (https://localhost:8443 v devu, proxováno přes Vite). Konfigurace přes Vite env proměnné VITE_OAUTH_CLIENT_ID, VITE_OAUTH_CLIENT_SECRET, VITE_OAUTH_SCOPE.
authorizedFetch (src/api/authorizedFetch.ts)
Wrapper přes fetch, který automaticky přidává Authorization: Bearer …, řeší token refresh a 401 redirect na login.
useAuthorizedQuery / useAuthorizedMutation (src/hooks/useAuthorizedFetch.ts)
React Query wrappery nad authorizedFetch. Použijte je pro vlastní HTTP volání mimo HAL navigation flow.
Pro typesafe volání známých endpointů použijte useKlabisApiQuery / useKlabisApiMutation z src/api/index.ts — generované typy z OpenAPI specifikace. Pro HAL navigation use cases ale typicky stačí useHalPageData().
