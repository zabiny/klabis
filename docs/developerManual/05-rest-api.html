REST API a HAL+FORMS

Veškeré REST endpointy vrací application/prs.hal-forms+json — HAL doplněný o popis affordancí (POST/PUT/PATCH/DELETE operací včetně metadat formuláře). Klabis poskytuje helpery klabisLinkTo a klabisAfford, anotaci @HalForms pro pole, PatchField<T> pro PATCH operace a centrální exception handler.

HAL+FORMS principy

  • Každý odpověďový resource má _self link a (pokud existují) další navigační linky.
  • Odkazy (_links) odkazují pouze na GET endpoints.
  • Affordances (_templates) popisují modifikační operace (POST/PUT/PATCH/DELETE) i s metadaty polí (typ, required, readOnly, options).
  • Frontend čte tyto metadata a generuje formuláře dynamicky (viz hal-navigator-patterns).

klabisLinkTo a klabisAfford

Wrapper kolem Spring HATEOAS linkTo() a afford(). Přidává:

  • Filtrování podle autorizace — pokud aktuální uživatel nemá oprávnění na cílovou metodu, link/affordance se nevygeneruje. Vrací se Optional<WebMvcLinkBuilder>.
  • Aplikace anotací @HalForms z record komponent na affordance metadata.
import static com.klabis.common.ui.HalFormsSupport.klabisLinkTo;
import static com.klabis.common.ui.HalFormsSupport.klabisAfford;

klabisLinkTo(methodOn(MemberController.class).getMember(memberId, null))
    .map(link -> link.withSelfRel()
        .andAffordances(klabisAfford(methodOn(MemberController.class)
            .updateMember(memberId, null, null))));
klabisLinkTo(invocation)
Vrátí Optional<WebMvcLinkBuilder> pokud má uživatel oprávnění na cílovou metodu, jinak Optional.empty().
klabisAfford(invocation)
Vrátí seznam Affordance objektů (prázdný, pokud uživatel nemá oprávnění). Aplikuje @HalForms anotace z body parametru (record components).
klabisAffordWithOptions(invocation, propertyOptions)
Stejné jako klabisAfford, ale navíc vloží inline options hodnoty pro zadané property (např. dropdown values).
entityModelWithDomain(dto, domain)
Místo EntityModel.of(dto) — wrapper, který doplní domain info do odpovědi (potřebné pro frontend pro identifikaci typu).

Anotace @HalForms

Použitelná na record komponentu DTO. Ovlivňuje, jak se pole zobrazí v HAL+FORMS metadatech. 

@Target(ElementType.RECORD_COMPONENT)
public @interface HalForms {
    enum Access { READ_ONLY, NONE, READ_WRITE, DEFAULT }
    Access access() default Access.DEFAULT;
    String formInputType() default "";   // např. "email", "tel", "date"
}
AccessVýznam
READ_ONLYPole je v metadatech, ale readOnly=true (zobrazí se, ale nelze měnit).
READ_WRITEPole je v metadatech, readOnly=false.
NONEPole se v metadatech vůbec neobjeví (skryto z formuláře).
DEFAULTVýchozí chování podle Spring HATEOAS — record komponenta = read-write.
 
public record CalendarItemDto(
        @HalForms(access = HalForms.Access.READ_ONLY) CalendarItemId id,
        String title,
        @HalForms(access = HalForms.Access.READ_ONLY) EventId eventId
) {}

RootController a navigace

/api vrací prázdný resource s linky do jednotlivých modulů. Linky nepřidává samotný RootController — místo toho každý modul registruje RepresentationModelProcessor<EntityModel<RootModel>> (typicky v {module}/infrastructure/restapi/), který doplní svoje top-level linky.

Frontend si při startu načte /api a podle dostupných linků postaví navigační menu — nepotřebuje si pamatovat URL strukturu. Modul, který nemá linky pro daného uživatele, prostě v menu nebude.

EntityModelWithDomain a ModelWithDomainPostprocessor

Wrapper přes EntityModel, který do odpovědi přidává atribut _domain s názvem byznys domény. Frontend podle něj rozpozná typ payload-u (např. "Member", "Event") nezávisle na URL. ModelWithDomainPostprocessor je Jackson postprocessor, který tento atribut serializuje.

PatchField — partial updates

Pro PATCH semantiku potřebujeme rozlišit "pole nebylo posláno" vs. "pole je explicitně null" (vymazat). PatchField<T> tuto distinkci umožňuje. 

public final class PatchField<T> {
    public static <T> PatchField<T> of(T value);
    public static <T> PatchField<T> notProvided();

    public boolean isProvided();
    public T patchValue(T originalValue);          // returns provided value or original
    public void ifProvided(Consumer<? super T> action);
    public <O> PatchField<O> map(Function<T, O> mapper);
    public T throwIfNotProvided();
}

Použití v request DTO:

public record UpdateMemberRequest(
        PatchField<String> firstName,
        PatchField<String> lastName,
        PatchField<String> email,
        PatchField<AddressRequest> address
) {}

V mapperu / aplikační vrstvě se aplikuje na agregát:

request.firstName().ifProvided(name -> member.changeFirstName(name));
String newEmail = request.email().patchValue(member.getEmail().value());
Jackson deserializer (PatchFieldDeserializer, registrován přes PatchFieldModule) rozpoznává, že JSON klíč chybí, vs. že je null. Modul je registrován automaticky.

@ValidOptionalSize — validace volitelných polí

Standardní @Size nefunguje na Optional<String>. @ValidOptionalSize je vlastní constraint v common.validation: 

public record AddressRequest(
        @ValidOptionalSize(max = 100) Optional<String> street,
        @ValidOptionalSize(max = 50) Optional<String> city
) {}

Exception handling

MvcExceptionHandler v common.mvc je @RestControllerAdvice, který mapuje doménové výjimky na HTTP 4xx/5xx s application/problem+json tělem (RFC 7807 ProblemDetail).

VýjimkaHTTP status
IllegalArgumentException, InvalidDataException, BusinessRuleViolationException400
AuthorizationException, InsufficientAuthorityException403
ResourceNotFoundException404
OptimisticLockingFailureException409
CannotRemoveLastOwnerException, DirectMemberAdditionNotAllowedException, …422
MethodArgumentNotValidException / HandlerMethodValidationException400 (s errors[])

V controlleru tedy nepoužíváme try/catch — vyhodíme typovanou doménovou výjimku a handler ji zformátuje. Vlastní výjimku přidáte přímo do MvcExceptionHandler.

Konfigurace

HateoasConfiguration (common.hateoas)
Globální Spring HATEOAS setup, registrace HalFormsMultiPropertyModule (rozšířená HAL+FORMS metadata pro více properties), FieldSecurityJacksonModule (filtrování polí podle autorizace).
MvcConfiguration (common.mvc)
WebMvc configuration — registrace argument resolverů (např. CurrentUserArgumentResolver pro @ActingUser) a interceptors.
SpaFallbackController (common.ui)
Vrací index.html pro všechny ne-API URL (SPA routing). Frontend bundluje React aplikaci.
@MvcComponent (common.mvc)
Stereotyp pro MVC komponenty (controllery, advice). Aktuálně rozšiřuje @RestController a slouží pro budoucí přizpůsobení component scanu / aspektů. Použití analogické jako @RestController.

Aktuální uživatel v controlleru

Pro získání přihlášeného uživatele používejte parametrové anotace @ActingUser a @ActingMember z modulu members. Nepoužívejte přímo SecurityContextHolder — ztratíte typovou bezpečnost a usnadníte si testování.

@ActingUser

Pro všechny přihlášené uživatele — i pro ty bez Member profilu (např. admin účty). 

@Target(ElementType.PARAMETER)
public @interface ActingUser {}
Typ parametruCo dostanete
UserIdUUID aktuálního uživatele.
CurrentUserDataPlná data: userName, userId, memberId (nullable), authorities.
 
@PostMapping("/members")
public ... registerMember(@RequestBody Request body, @ActingUser UserId currentUserId) { ... }

@PatchMapping("/members/{id}")
public ... updateMember(@PathVariable MemberId id,
                        @RequestBody Request body,
                        @ActingUser CurrentUserData currentUser) { ... }

@ActingMember

Vyžaduje Member profil. Pokud uživatel nemá Member (např. admin), vyhodí MemberProfileRequiredException ještě před vstupem do controller metody (handler vrátí HTTP 403). 

@Target(ElementType.PARAMETER)
public @interface ActingMember {}

Podporované cílové typy: MemberId.

@PostMapping("/events/{id}/registrations")
public ... register(@PathVariable EventId id,
                    @ActingMember MemberId actingMember) { ... }

CurrentUserData

public record CurrentUserData(
        String userName,
        UserId userId,
        @Nullable MemberId memberId,
        Set<Authority> authorities) {

    public boolean isMember();
    public boolean hasAuthority(Authority authority);
    public boolean isMemberOf(Predicate<MemberId> membershipCheck);
}
Pro autorizaci v rámci kontextu metody preferujte @HasAuthority nebo @OwnerVisible (kap. 06). currentUser.hasAuthority(...) v těle metody používejte jen pro obsahové rozhodování (např. které pole vyplnit), ne pro security checky.

CurrentUserArgumentResolver

Spring MVC HandlerMethodArgumentResolver v members.infrastructure.authorizationserver, který implementuje obě anotace. Načítá data z KlabisJwtAuthenticationToken (kap. 06) a v případě CurrentUserData doplní username a authorities. Auto-registruje se přes WebMvcConfigurer.
