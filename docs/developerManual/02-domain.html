Doménová vrstva

Stavební bloky pro doménový model: KlabisAggregateRoot (base třída agregátu), AuditMetadata (audit pole), type-safe ID jako value object, doporučení pro hodnotové typy a domain events, a EncryptedString pro citlivé údaje.

KlabisAggregateRoot

Každý agregát dědí z KlabisAggregateRoot<A, ID>, který rozšiřuje Spring Data AbstractAggregateRoot o audit metadata a identity-based equality. 

public abstract class KlabisAggregateRoot<A extends KlabisAggregateRoot<A, ID>, ID>
        extends AbstractAggregateRoot<A> {

    public abstract ID getId();
    public AuditMetadata getAuditMetadata();
    public void updateAuditMetadata(AuditMetadata metadata);

    @DomainEvents
    public List<Object> getDomainEvents();   // pro Spring Modulith
    public void clearDomainEvents();

    // equals/hashCode podle getId()
}
Self-typed parametr A extends KlabisAggregateRoot<A, ID> umožňuje fluent API z AbstractAggregateRoot vracet konkrétní typ agregátu.

Použití

public class Member extends KlabisAggregateRoot<Member, MemberId> {

    public static Member registerNewMember(RegisterMember command) {
        Member member = new Member(...);
        member.registerEvent(MemberCreatedEvent.fromAggregate(member));
        return member;
    }
}
registerEvent(event)
Zapíše domain event do agregátu. Spring Data ho po save() publikuje a Spring Modulith uloží do transactional outboxu (viz kap. 08).
updateAuditMetadata(metadata)
Volá persistence vrstva po načtení/uložení. Aplikační kód toto nevolá.
equals/hashCode
Porovnává se podle getId(), ne podle stavu — což je správně pro DDD agregát.

AuditMetadata

@RecordBuilder
@ValueObject
public record AuditMetadata(
        Instant createdAt, String createdBy,
        Instant lastModifiedAt, String lastModifiedBy,
        Long version) {

    public AuditMetadata markModified(String modifiedBy);
    public AuditMetadata incrementVersion();
}

Vyplňuje persistence vrstva — auditor (createdBy/lastModifiedBy) je extrahován ze SecurityContext (registrationNumber přihlášeného uživatele) v JdbcConfiguration.

Type-safe identifikátory

Každý agregát má vlastní typ ID — record implementující jMolecules Identifier. Eliminuje záměnu UUID různých entit. 

@ValueObject
public record MemberId(UUID value) implements Identifier {
    public UUID uuid() { return value; }
}
Pokud má aggregát relaci 1:0–1 s jiným (jako Member ↔ User), používá se shared ID: oba agregáty sdílí stejnou UUID hodnotu (MemberId.toUserId() / MemberId.fromUserId(...)). Detail viz aggregate-checklist v skillu backend-patterns.

Value objects

Hodnotové typy (Address, EmailAddress, BirthNumber, …) jsou imutabilní recordy v {module}/domain/, anotované @ValueObject. Validace proběhne v compact konstruktoru: 

@ValueObject
public record EmailAddress(String value) {
    public EmailAddress {
        Assert.hasText(value, "email is required");
        Assert.isTrue(value.contains("@"), "must contain @");
    }
}

Pro builder pattern lze přidat @io.soabase.recordbuilder.core.RecordBuilder.

Domain events

Eventy jsou recordy v kořenovém package modulu (jsou součástí veřejného API): 

public record MemberCreatedEvent(MemberId memberId, EmailAddress email, Instant occurredAt) {
    public static MemberCreatedEvent fromAggregate(Member member) { ... }
}

Agregát eventy registruje pomocí registerEvent(event). Spring je publikuje po úspěšném commitu transakce. Listenery v jiných modulech viz kap. 08.

Repository (interface)

Repository interface patří do domain/ a deklaruje pouze doménové operace. Implementace (JDBC adapter) je v infrastructure/jdbc/ — viz kap. 04. 

@org.jmolecules.architecture.hexagonal.SecondaryPort
public interface MemberRepository {
    Optional<Member> findById(MemberId id);
    Member save(Member member);
}

EncryptedString — GDPR-citlivá pole

Value object pro řetězce, které musí být šifrované at-rest. Spring Data JDBC konvertery zajistí transparentní šifrování/dešifrování (Jasypt). toString() vrací "***", aby hodnota neunikla do logů. 

public record EncryptedString(String value) {
    public static EncryptedString of(String value);
    @Override public String toString() { return "***"; }
}

Použití v doméně — pole, které má být šifrováno, má v memento typ EncryptedString; konvertery jsou registrované v JdbcConfiguration. Manuální šifrování (mimo JDBC) přes EncryptionService.

Příklady v repu: Member používá EncryptedString pro BirthNumber v memento vrstvě (viz members/infrastructure/jdbc/MemberMemento). Audit access k citlivému poli se loguje pomocí BirthNumberAccessedEvent.
