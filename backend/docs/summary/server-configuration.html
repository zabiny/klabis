Konfigurace serveru

Zabezpečená komunikace přes HTTPS a HTTP/2 protokol s certifikáty SSL/TLS pro vývojové, testovací i produkční prostředí.

HTTPS pouze na portu 8443

Systém vyžaduje šifrovanou komunikaci přes HTTPS protokol na portu 8443. HTTP na portu 8080 není podporován. Všechny endpointy včetně OAuth2 používají výhradně HTTPS.

Hlavní funkce

HTTPS Protokol

Vynucené HTTPS na portu 8443 ve všech profilech (dev, test, prod). HTTP port 8080 není podporován.

HTTP/2 Podpora

Automatická podpora HTTP/2 pro lepší výkon díky multiplexingu a kompresi hlaviček.

SSL Certifikáty

Profil-specifické konfigurace keystore pro vývoj, testování i produkci s environment proměnnými.

OAuth2 přes HTTPS

Všechny OAuth2 URL včetně autorizace, tokenů a redirect URIs používají HTTPS protokol.

Bezpečnost TLS

Minimální verze TLS 1.2+, bezpečné šifrovací sady a HSTS hlavička pro maximální zabezpečení.

Generování certifikátů

Nástroje a dokumentace pro generování self-signed certifikátů pro lokální vývoj.

TLS/SSL konfigurace

Minimální verze TLS

Systém přijímá pouze TLS 1.2 a vyšší. TLS 1.0 a 1.1 jsou odmítnuty. Všechny spojení používají silné šifrovací sady s forward secrecy.

Profil-specifické certifikáty

Vývojový profil

Self-signed certifikát s aliasem dev-klabis. Generován lokálně pro localhost.

Testovací profil

Testovací certifikát v repo pro CI/CD. Automaticky dostupný v integračních testech.

Produkční profil

Certifikát z environment proměnných: SSL_KEYSTORE_PATH, SSL_KEYSTORE_PASSWORD, SSL_KEY_ALIAS.

HTTPS URL v celém systému
  • OAuth2 issuer URL: https://localhost:8443
  • JWKS endpoint: https://localhost:8443/.well-known/jwks.json
  • Redirect URIs: https://localhost:8443/auth/callback.html
  • E-mailevé linky: https://localhost:8443/auth/password-setup.html
  • OpenAPI dokumentace: https://localhost:8443 Swagger UI
  • CORS origins: https://localhost:8443 povolen
Generování vývojového certifikátu

Lokální vývoj

Pro lokální vývoj použijte poskytnutý skript pro generování self-signed certifikátu. Certifikát je platný 365 dní pro localhost, 127.0.0.1 a ::1.

Bezpečnostní hlavičky

HTTP Strict Transport Security

HSTS Strict-Transport-Security max-age=31536000; includeSubDomains

Všechny HTTPS odpovědi obsahují HSTS hlavičku s roční platností pro vynucení HTTPS v prohlížečích.

Přechod z HTTP na HTTPS

Migrační checklist

1. Generovat SSL certifikát pro vývoj
2. Důvěřovat certifikátu v prohlížeči a OS
3. Aktualizovat environment proměnné
4. Aktualizovat OAuth2 klient konfiguraci
5. Přepsat všechny URL z http:// na https://
