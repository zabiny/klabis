Událostmi řízená architektura

Event-driven architektura pro asynchronní zpracování operací s transakční bezpečností a modulárním designem pomocí Spring Modulith.

Spring Modulith

Modulární monolit s jasně definovanými hranicemi modulů (users, members, email) a asynchronní komunikací přes domain events.

Principy architektury

Domain Events

Agregáty publikují domain events při změnách stavu. Eventy nesoucí business význam.

Transakční safety

Eventy se zpracovávají až po commitu transakce. Žádné "ztracené" e-maily při rollbacku.

Asynchronní zpracování

Event handlers běží asynchronně, neblokují hlavní transakci. Lepší výkon a UX.

Modulární design

Jasně definované hranice modulů. Volné spojení přes events, silná koheze uvnitř modulů.

Transactional outbox

Event persistence v rámci transakce. Záruka doručení i při výpadcích.

Observability

Event registry, modul detection, integrační testy modulů. Transparentní architektura.

Moduly systému
  • Users module: Správa uživatelských účtů, OAuth2, oprávnění, password setup flow
  • Members module: Registrace a správa členů, registrační čísla, osobní údaje
  • Email module: Odesílání transakčních e-mailů, šablony, SMTP komunikace
  • Komunikace: Moduly spolu komunikují přes domain events (member-created, user-created, atd.)
Typy eventů

MemberCreatedEvent

Při registraci nového člena. Triggeruje uvítací e-mail a password setup flow.

UserCreatedEvent

Při vytvoření User entity. Triggeruje password setup token generování.

PermissionsChangedEvent

Při změně uživatelských oprávnění. Triggeruje audit log.

PasswordSetEvent

Při úspěšném nastavení hesla. Invaliduje password setup token.

Transactional Event Listener

Spring @TransactionalEventListener

Event handlery jsou označeny @TransactionalEventListener(phase = AFTER_COMMIT).

Tím je zaručeno, že event handler běží až po úspěšném commitu transakce. Pokud se transakce rollbackuje, event handler se nevyvolá.

Výhody architektury
  • Oddělení zájmů: Email logika není v Member agregátu. Čistší domain model.
  • Testovatelnost: Moduly lze testovat izolovaně. Integrační testy pro event flows.
  • Škálovatelnost: Event handlers lze škálovat nezávisle (future: async processing).
  • Resilience: Selhání jednoho handleru neovlivní ostatní. Retry mechanismy.
  • Transparentnost: Spring Modulith poskytuje vizualizaci event flows.
Best Practices

Principy implementace

Immutable events: Eventy jsou neměnné (Java records).
Event naming: Past tense (MemberCreated, UserUpdated).
Event payload: Obsahuje ID agregátu a relevantní data.
Handler idempotence: Handlery by měly být idempotentní pro replay.
Logging: Všechny eventy se logují pro debugging a audit.
